# セットアップガイド

## GitHubへのプッシュ手順

### 1. GitHubリポジトリの作成

1. GitHubにログインして、新しいリポジトリを作成してください
2. リポジトリ名は任意（例: `minima-order-app`）

### 2. リモートリポジトリの追加とプッシュ

```bash
# リモートリポジトリを追加（YOUR_USERNAMEとYOUR_REPO_NAMEを置き換えてください）
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git

# メインブランチにプッシュ
git push -u origin main
```

既にリモートリポジトリが設定されている場合は：

```bash
git remote -v  # リモートリポジトリを確認
git push -u origin main
```

## スマホからアクセスする設定

### 自動設定（完了済み）

✅ Next.jsの開発サーバーが`0.0.0.0`でリッスンするように設定済み
✅ ローカルネットワークからのアクセスが可能

### 手動で行うこと

#### 1. 開発サーバーの起動

```bash
npm run dev
```

起動時に以下のような表示が出ます：

```
▲ Next.js 16.0.1 (Turbopack)
  - Local:        http://localhost:3000
  - Network:      http://172.16.7.246:3000
```

#### 2. スマホからアクセス

1. **PCとスマホを同じWi-Fiネットワークに接続**
2. **スマホのブラウザでNetworkアドレスにアクセス**
   - 例: `http://172.16.7.246:3000`
   - 表示されたNetworkアドレスを使用してください

#### 3. ファイアウォールの設定（必要に応じて）

macOSの場合：

```bash
# システム環境設定 > セキュリティとプライバシー > ファイアウォール
# Node.jsを許可する必要がある場合があります
```

#### 4. Firebase Consoleでの設定（オプション）

ローカルネットワーク経由でのアクセスでは、通常Firebaseの追加設定は不要です。
ただし、本番環境でデプロイする場合は、Firebase Consoleで許可されたドメインを追加する必要があります。

**Firebase Consoleでの設定手順：**

1. [Firebase Console](https://console.firebase.google.com/)にアクセス
2. プロジェクト「minima-order」を選択
3. 左メニューから「Authentication」を選択
4. 「設定」タブ > 「承認済みドメイン」を確認
5. 必要に応じて、デプロイ先のドメインを追加

**注意**: 開発環境（localhost）は自動的に許可されています。

### カメラ機能について

スマホでQRコード読み取り機能を使用する場合：

- **HTTPSが必要な場合があります**
- ローカルネットワーク経由（HTTP）でも動作する場合がありますが、ブラウザによって異なります
- 動作しない場合は、HTTPSを使用するか、テーブル番号の手入力機能を使用してください

### トラブルシューティング

#### スマホからアクセスできない場合

1. **PCとスマホが同じWi-Fiネットワークに接続されているか確認**
2. **PCのファイアウォールがポート3000をブロックしていないか確認**
3. **Networkアドレスが正しいか確認**（開発サーバー起動時に表示されるアドレス）
4. **PCのIPアドレスを確認**:
   ```bash
   ifconfig | grep "inet " | grep -v 127.0.0.1
   ```

#### Firebase接続エラーが発生する場合

- ローカルネットワーク経由では、通常Firebaseは正常に動作します
- エラーが発生する場合は、Firebase Consoleでネットワーク設定を確認してください
